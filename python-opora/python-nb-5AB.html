Implicitní hodnota parametru¶

Definice funkce obsahuje konstrukci, kterou zatm neznme tzv. implicitn hodnotu parametru (v naem ppad m parametr length implicitn hodnotu 3). Dky pouit implicitn hodnoty lze funkci volat bu s jednm parametrem (promnn length m pak hodnotu tohoto parametru) nebo bez parametru (promnn lengthm pak implicitn hodnotu 3).

Pro implicitn hodnoty plat nsledujc omezen:

  • m-li parametr implicitn hodnotu, pak ji mus mt i vechny nsledujc (jinak by nebylo pozin piazen parametr jednoznan)
  • implicitn hodnoty nesm vyuvat hodnoty jinch parametr (jen konstanty, globln promnn, apod.)
  • implicitn hodnoty by mly bt nemnnmi (tj. sla a etzce jsou OK, seznamy nikoliv)

Posledn dv omezen jsou dsledkem skutenosti, e k vyhodnocen implicitnch hodnot dochz jen jednou v mst definice funkce, nikoliv pi kadm jejm vyvoln. V tomto okamiku nejsou dostupn dn dal parametry (funkce jet nebyla volna) a vznikl objekt je sdlen vemi volnmi (co me u mnitelnch objekt vst k podivnmu a neoekvanmi chovn).

Uvnit tla funkce je pouita metoda string.join, kter spojuje seznam etzc do jedinho velkho etzce. Navc vkld mezi spojovan etzce oddlova (co je fixn znak nebo skupina znak). Metoda join je tedy opakem metody split (ta naopak podle oddlovae etzec rozdluje). Adrestem metody je etzec oddlovae (zde je to przdn etzec take etzce budou spojeny bez oddlovae). Parametrem je seznam nhodnch slabik (seznam je vytvoen komprehenz pomoc length voln na funkce randomSyllable).

def rotate(lst, shift=1):  # shift = o kolik pozic se seznam posune
    shift %= len(lst)
    if shift == 0:
        return lst # vracme nezmnn originln seznam
    lst.extend(lst[:shift]) # pidme na konec kopii prvk s indexem 0 a shift-1 (vetn)
    del lst[:shift] # a originln prvky smaeme (maeme `shift` prvk)
    return lst

def all_inline(planets, ephem_time, sector_angle): 
    """
        planets: seznam ephem objekt planet zahrnutch do vpotu
        ephem_time: as vpotu (v ephem representaci)
        sector_angle: hel vsee do n se mus vjt vechny planety (ve stupnch)
    """
    # nejdve pepoteme limitn hel 
    sector_angle = math.radians(sector_angle)
    # pak musme vechny objekty planet aktualizovat (pepotat) pro dan datum
    for planet in planets:
        planet.compute(ephem_time)
    # a nyn u budeme kontrolovat vechny mon dvojice
    for planet1 in planets: # vnj cyklus
        for planet2 in planets: # vnitn cyklus
            if angle_between(planet1, planet2) > sector_angle: # tato dvojice se nevejde do vsee
                return False # u nemusme dl hledat 
    # po ukonen cyklu je zejm, e planety se do vsee vejdou (i ty nejvzdlenj maj mezi sebou men hel)
    return True

Rekurze¶

Jednm z mechanism programovn, kter zce souvisej s funkcemi (obecn podprogramy) je rekurze.

Princip rekurze se pouv i v matematice, tam je to vak relativn okrajov tma.

Podvejme se napklad na jednu z monch definic faktorilu:

$n!=n\cdot (n-1)!$

$1! = 1$

Vimnte si, e v tto definici je faktoril sla $n$ definovn pomoc jinho faktorilu (sla $n-1$). Napklad faktoril 3! je podle tto definice roven $3\cdot 2!$. Pokud pouijeme definici znovu zskme hodnotu $3\cdot 2\cdot 1!$, co po dosazen podle druhho vztahu vede na $3\cdot 2\cdot 1 = 6$.

Tento tzv. rekurzivn vztah vede k vnoen aplikaci rekurentnho pravidla, kter vak skon po konenm potu krok pouitm tzv. koncovho ji nerekurentnho pravidla.

Tuto definici lze velmi snadno realizovat i v Pythonu:

def factorial(n):
    return 1 if n== 1 else n*factorial(n-1) # pro procvien podmnn vraz

Out[34]:

Rekurzi snadno poznte podle toho, e se v tle funkce vol tat funkce (s jinou hodnotou parametru), tj. funkce vol sebe samu.

Pouit takto jednoduchch rekurz v Pythonu vak vede ve vtin ppad k znan neefektivnmu kdu. Napklad faktoril lze mnohem efektivnji naprogramovat pomoc cyklu.

Nkdy me vst pouit rekurze k extrmn pomalmu kdu. Klasick (proti)pklad je rekurzivn vpoet Fibonnaciho posloupnosti:

$F(n)=\left\{{\begin{matrix}1&&{\mbox{pro }}n\in{0,1};\ \ \,\\F(n-1)+F(n-2)&&{\mbox{jinak.}}\end{matrix}}\right.$

Jinak eeno n-t Fibonnacciho slo je soutem dvou pedchozch len posloupnosti, piem posloupnost zan dvmi jednikami. Dalmi leny posloupnosti je 2, 3, 5, 8, 13, atd.

I tuto rekurzivn definici lze pmo zapsat v Pythonu:

def fastindex(s, wanted, leftindex, rightindex):
    """
     s: prohledvan etzec
     wanted: hledan prvek
     leftindex: index poten (od nj se hled)
     rightindex: index (a po nj vetn se hled)
    """
    # print(leftindex, rightindex) # jen pro ely ladn (zkuste odpoznmkovat)
    if rightindex < leftindex:  # klov koncov podmnka (v ppad neexistence prvku)
        return -1
    left = s[leftindex] # poloka na levm indexu (prvn = minimln)
    right = s[rightindex] # poloka na pravm indexu (posledn = maximln)
    if wanted < left or wanted > right: # hledan hodnota le mimo rozsah
        return -1  # neme bt tedy nalezena 
    interindex = leftindex + (rightindex-leftindex) * (wanted - left) // (right - left)
    if s[interindex] == wanted:  # prvek nalezen
        return interindex
    if s[interindex] < wanted:  # prvek je v pravm segmentu
        return fastindex(s, wanted, interindex, rightindex-1)
    else:                       # resp. v levm
        return fastindex(s, wanted, leftindex+1, interindex)

Klovou st tla funkce je vpoet odhadovan pozice indexu, pomoc jednoduch linern interpolace. Pokud znme hodnotu prvnho prvku (s indexem leftindex) a hodnotu prvku poslednho (index rightindex), pak pi pedpokladu rovnomrnho rstu prvk meme odhadnout pozici hledanho prvku jednoduchou interpolac (viz obrzek).

Interpolan seka spojuje body s pozic (leftindex, left) a (rightindex, right). Pokud znme tuto seku nen problm najt pro hledanou hodnotu wanted index, kter j podle interpolan seky pslu. Tento index ozname jako interindex.

Vimnte si, e pokud je hledan prvek roven prvnmu (levmu) je vsledek interpolanho vrazu skuten roven levmu indexu (leftindex + 0 / (right - left)), stejn tak je roven pravmu indexu pi hodnot (wanted == right).

Pokud interdidex indexuje hledan prvek (s[interindex] == wanted) pak tento index vrtme. Jinak se provede rekurzvn voln. V kterm vak budou pouit jin lev a prav index. Pokud je hledan hodnota v lev sti, pak bude vnoen voln hledat mezi indexy leftindex+1 a interindex (vetn), v ppad prav sti mezi indexy intertindex a rightindex (opt vetn). V naem ppad lze toto vnoen hledn znzornit dalm obrzkem.

Klovou roli m posunut levho resp. pravho okraje na nsledujc (leftindex+1) resp. pedchzejc poloku (rightindex-1). Tm se prohledvn nejen o nco urychl (je zbyten opt kontrolovat krajn poloku), ale pedevm se zabrn vzniku nekonen rekurze (tj.do potenciln nekonen vnoen voln, kter v realit vedou po krtk dob k pedasnmu ukonen programou vjimkou). I tento drobn posun toti zajituje, e nov interval je vdy men ne pedchoz (a to i tehdy pokud interindex padne do nkterho z pvodnch krajnch). Protoe v kadm kroku zmenujeme konen interval index, je zejm, e voln mus nkdy skonit (tj. je dosaeno stavu rightindex < leftindex).

Tato situace nastv i v naem ppad, kde vpoet interindexu vede k hodnot 5 (vsledek by ml bt cca 5,6, ale dky pouit celoslenho dlen se zaukrouhl na 5). Interindex je tak roven pvodnmu levmu indexu.
