Objektově orientované programování¶

Python je jak ji bylo eeno multiparadigmatick programovac jazyk tj. podporuje vce rznch styl programovn. Pesto vak jen jedno z tchto tzv. paradigmat ovlivuje Python na vech rovnch programovn objektov orientovan paradigma (OOP).

Ve svt tohoto paradigmatu existuj tzv. objekty, kter maj svou jedinenou identitu, mn sv stavy, vznikaj a zanikaj. Pedevm vak spolu komunikaj prostednictvm voln metod ze svch rozhran (rozhran je mnoina metod, kter objekty nabzej ostatnm objektm).

Objekty se navc seskupuj do td. Tda obsahuje objekty se stejnm rozhranm, tj. objekty, kter se pi komunikaci chovaj podobn (resp. stejn, pokud maj identick vnitn stav).

S objekty a tdami jsme se ji setkali. sla jsou v Pythonu velmi jednoduch objekty, kter jako sv rozhran nabzej bn aritmetick operace. Zkladnmi selnmi tdami jsou tdy int (representace celch sel) a float (representace racionlnch sel pomoc tzv. pohybliv dov rky). Mezi jednoduch objekty pat i objekty tdy bool (jsou jen dva True a False).

O nco sloitj jsou objekty tdy string oznaovan jako etzce. Ty nabzej ve svm rozhran vt poet metod. S selnmi objekty vak sdlej dv podstatn a vzjemn provzan vlastnosti: jsou nemnn a a z vnjho pohledu existuje vdy jen jeden objekt se stejnm stavem (nap. z vnjho pohledu existuje jen jeden objekt sla 2). I kdy intern me existovat vce kopi stejnho objektu, jsou tyto objekty nerozliiteln.

To ji neplat pro seznamy a mnoh dal sloitj objekty. Ty svj stav v prbhu svho ivota mn, a jejich identita je nezvisl na jejich obsahu.

Representace časových údajů¶

I kdy se to na prvn pohled nezd, je representace asovch daj jednou z nejkomplikovanjch problm tvrc standardnch knihoven. Dvodem je relativn komplikovan kalendn systm a jet komplikovanej systm asovch psem s jejich stlmi posuny a lenmi pravidly pro jejich kadoron posuny (tzv. letn as). Proti tomu jsou problmy dan vkldnm pestupn sekundy jen detail (me to vak znan zkomplikovat program oekvajc milisekundovou pesnost).

Ve skutenosti se ukazuje, e dn univerzln pouiteln representace asu neexistuje. Proto i ve standardn knihovn Pythonu jich existuje nkolik, a mnoh dal pidvaj knihovny tetch stran (jako nap. pyephem).

Zkladn a veobecn podporovanou representaci nabz standardn modul datetime. Tato implementace se zamuje na podporu representace asu v modern dob a modernm gregorinskm kalendi (tj. cca po prvn svtov vlce) s pesnost na rovni minut i destek vtein. Hod se tak pro bn historick a prbn databzov zznamy (datumy narozen, asy prodej apod.) a ekonomickou statistiku. Nepodporuje vak sprvn asov psma. Nezohleduje toti posuny as v rznch mstech jako je zmna psmovho asu (v nkterch zemch se to dje i nkolikrt v desetilet) a pedevm letn as.

Jednodu rozhran nabz knihovna pro ty aplikace, kter se nemus starat o asov psma a vyuit letnho asu.

def addMonths(datum, months):
    """
        datum: pvodn datum
        months: poet msc, kter maj bt piteny 
    """
    # zskme jednotliv sti data
    monthDays = [31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]
    d = datum.day
    m = datum.month - 1 # msce jsou potny od nuly
    y = datum.year
    m = (m + months) % 12 # piteme msce (a upravme do rozsahu 0-11) 
    y = y + (m + months) // 12 # k roku musme pist bloky 12 msc
    if y % 4 == 0:  # je-li rok pestupn (zjednodueno)
        monthDays[1] = 29  # pak upravme dlku msce nora (index=1)
    d = min(d, monthDays[m])  # upravme dny na konci msce 
    # (denn slo nesm bt vt ne poet dn v msci)
    return datetime.date(y, m+1, d)

Práce se soubory (proudy dat)¶

Jakkoliv plnohodnotn program mus pracovat s externmi daty (vypsn "Hello, world" i vpis jakhokoliv jinho fixnho textu nen programem). Extern data jsou typicky zadvna interaktivn uivatelem (viz nm ji znmou vestavnou funkci input), tena z datovch soubor nebo jsou zskvana z Internetu.

Vechny tyto extern zdroje dat lze sjednotit do abstraktnho mechanismu proudu dat (angl. stream). Nejebecnj typem proudu jsou bytov proudy, kter poskytuj konenou posloupnost byt (byte = binrn representace sel 0-255 = 8 bit).

Krom bytovch proud se pouvaj i proudy znakov. Znakov proudy meme chpat jako konenou posloupnost znak (psmen, slic, symbol, apod.). Intern se kad znak ukld i pen pomoc jednoho i vce byt. kter kduj pozici znak v njak znakov sad (tabulce jednotlivch znak).
